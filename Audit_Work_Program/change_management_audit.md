# Change Management Audit Program

## 1. Objective
Determine whether system changes follow a controlled, approved process.

## 2. Risks
- Unauthorized or untested changes.
- Outages resulting from poor change coordination.
- Lack of audit trail for system modifications.

## 3. Controls to Test
- CAB approval documentation.
- Ticket workflow and traceability.
- Segregation of duties.
- Emergency change controls.
- Rollback procedures.

## 4. Test Procedures
1. Pull change tickets for the past 3 months.
2. Select sample of 10 changes and validate:
   - Approval received.
   - Testing evidence attached.
   - Implementation notes documented.
   - Post-implementation review completed.

3. Review emergency change log.

## 5. Evidence Required
- CAB minutes
- Tickets from ITSM tool
- Change logs on systems
- Emergency change register
