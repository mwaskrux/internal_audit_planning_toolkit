# Backup & Disaster Recovery Audit Program

## 1. Objective
Assess whether backup processes and DR readiness provide adequate resilience.

## 2. Risks
- Irrecoverable data loss.
- Backup corruption.
- Failure to meet RPO/RTO.
- Unverified restore capabilities.

## 3. Controls to Test
- Backup job configuration.
- Retention policies.
- Backup encryption.
- DR plan existence and testing frequency.
- Restore test results.

## 4. Test Procedures
1. Review backup schedules and confirm coverage of critical data.
2. Verify backup job success/failure logs.
3. Confirm backups are encrypted in transit and at rest.
4. Evaluate off-site/cloud backup availability.
5. Review most recent restore test results.

## 5. Evidence Required
- Backup logs
- DR plan document
- DR test report
- Storage inventory
- Cloud backup console screenshot
